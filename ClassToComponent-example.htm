QVTR-XSLT

Model Transformation with QVTR and XSLT

Class to Component transformation

  

The example demonstrates how to implement an in-place transformation in the QVTR-XSLT tool.  An in-place transformation means the source model and the target  model is the same one at the time of the transformation running. The basic idea is that we specify the changed part of a model, and then merge the changed part into the original source model.
 

To do that, every class in the metamodel should have two attributes: a "targetId" and a "xmiDiff". The "targetId" indicates the ID of the source model element which will going to be changed. And the "xmiDiff" is an enumeration of changing operators, they are: "insertTo", "insertNext", "insertBefore", "replace" and "remove". The "insertTo" operator will insert the element as a child of targetId element, and the other operators will insert before or after the targetId  element, replace or remove the targetId element.

The class to component transformation example generates a component  from a class in rCOS model structure. It gets a class by name (though the function GetToComClass ) from the class model, and produces a component in the component model. The following operations described in an abstract way are performed  by the transformation:

  1. clz = getClassByName();
  2. com = createEmptyComponent();
  3. com.setName("Com_From_", clz.Name);
  4. iface = createEmptyInterface();
  5. iface.setName("Inter_From_"+clz.Name);
  6. iface.addOpAndParaFrom(clz);
  7. reali = createEmptyInterfaceRealization();
  8. reali.setSupplier(iface);
  9. reali.setClient(com);
  10. reali.setImplementingClass(clz);
  11. com.setinterfaceRealization(reali);
