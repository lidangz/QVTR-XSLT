rCOS

Refinement of Component and Object Systems

UML Sequence Diagrams to CSP transformation

  

Details of the transformation can be found here.

The transformation is specified as twelve QVT relations with their respective diagrams :

QVTR model    

 Metamodel: UML sequence diagrams  (SeqD).


 

Metamodel: CSP

relation: InteractionToCSP, the top-level relation which creates the root element of a CSP model.


  

relation : MessageToChannel, creating a CSP channel for each message.


       

relation: LifelineToProcess,  creating a CSP process for each lifeline.


       

relation: OperandToSub, preparing the fragment belonged to a interaction operand


       

  relation: ReceiveEventToPrefix


       

  relation: SendEventToPrefix

relation AltToExt

relation AltToIf

relation LoopToIf

relation OptToIf

relation BreakToIf

relation LifelineToPara, building a CSP system by parallel composition of CSP processes from lifelines

generated XSLT stylesheet   (include XSLT file)

input model :  A example sequence diagram

output CSP model

channel o1_o2_m1, o1_o2_m2, o1_o2_m3_rt, o2_o1_m3, o2_o1_m4, o2_o1_m1_rt

O1=O1_451
O1_451=o1_o2_m1!->O1_465
O1_465=if (c1) then o1_o2_m2!->O1_499 else O1_549
O1_499=if (c2) then o2_o1_m3?->o1_o2_m3_rt!->O1_465 else o2_o1_m4?->O1_465
O1_549=o2_o1_m1_rt?->SKIP

O2=O2_452
O2_452=o1_o2_m1?->O2_465
O2_465=if (c1) then o1_o2_m2?->O2_499 else O2_548

O2_548=o2_o1_m1_rt!->SKIP

system=O1 [|{|o1_o2_m1, o1_o2_m2, o1_o2_m3_rt, o2_o1_m3, o2_o1_m4, o2_o1_m1_rt|}|] O2
