UML to RDBMS transformation

  

The example is a slight variation on the classic ”simple UML model to simple RDBMS transformation” taken from QVT specification. This transformation is usually used to demonstrate the concepts and capabilities of transformation languages.;

The requirement for the transformation is as follows:

  • Each package in the UML model corresponds to a schema in the RDBMS model;

  • Each non-abstract class belonging to the package maps to a table with a primary key and an identifying column in the corresponding schema; 

  • Attributes with a primitive data type, both owned by a class itself and inherited from the class hierarchy, are translated into columns of    corresponding table; 

  • An attribute of a complex data type maps to a set of columns corresponding to all attributes of the class the complex data type referred to, prefix the names of the columns with the name of the referred class, and 

  • An association between two non-abstract classes is mapped to a foreign key relationship between the corresponding tables.
