QVTR-XSLT

Model Transformation with QVTR and XSLT

Getting Start with QVTR-XSLT

  

QVTR-XSLT  is a model transformation tool that provides support for QVT Relations in graphical notation and automatic transformation execution. It  consists of  a QVT Relations graphical editor and a code generator.  The graphical editor  supports the design of  QVT transformations by defining  metamodels, specifying QVT rules as UML object diagrams with OCL expressions, and validating the the transformations. The code generator translates the transformations  into executable XSLT programs

The QVTR-XSLT tool supports transformation parameters, transformation inheritance through rules overriding, and multiple input and output models. Furthermore, in-place  transformations are defined as modification (insert, remove, replace) of the existing model elements.

What inside the tool package

The follwoing files and directories are included in the tool archive:

  • QVTRtoXSLT.jar :  The XSLT code generator.

  • QVTR_Profile.mdzip:  The QVTR UML profile.

  • QVTRelation Diagram descriptor.xml : The toolbar definition file for the QVTR editor

  • XSLTrunner.jar : The transformation runner.

  • my_XMI_Merge.xslt: Additional common functions for transformation execution. This file may need to be put in the same directory with the generated XSLT files.

  • UMLtoDB_example : The directory contains files for the famous UML to RDBMS transformation case study, including:

                ▪  UMLtoDB-transfor.xml :  The transformation model file.

                ▪  library.mdzip :  The example UML model prepared in MagicDraw UML

                ▪  library.uml :  The example UML model exported in Eclipse UML 2 (v3.x)  XMI format

  • SimpleUMLtoSimpleRDBMS : Another UML to RDBMS case study, for comparision with Medini QVT:

                ▪  SimpleUMLtoSimpleRDBMS.xml :  The transformation model file.

                ▪  source.xmi :  An example input model (ecore format)

                ▪  umlTOrdbms_trace.xslt,  umlTOrdbms_Notrace.xslt :  Generated XSLT stylesheets, with and without trace output

                ▪  target_QVTR.xmi, target_medini.xmi :  The target models, from QVTR-XSLT and Medini QVT.

                ▪  trace_QVTR.xml, trace_medini.xml:  The trace models, from QVTR-XSLT and Medini QVT.

                ▪  SimpleUML2RDBMS_Medini :  The Medini QVT project for the case

    • Use the QVTR Graphical Editor

    To use the QVTR grphical editor, MagicDraw UML v16.5 or later is required.  Open the example tranformation file UMLtoDB-transfor.xml  with MagicDraw, when the profile is asked, select the QVTR_Profile.mdzip file. Then you can see the interface of the editor:

     

    You can design metamodels,  define QVTR transformations/relations/queries/functions  using the editor.  To validate you design,  press the √ button in upper-left , then select the "QVTR_Validate" for Validation Suite, press the "Validate" button.

    If f you have made some changes to the model, save the project as xml file.

    Generate XSLT Code

    If necessary, download and install recent  version  of   JRE (Java Runtime Environment) or  JDK ( Java SE Development Kit ) first.  (JDK) for your platform first.

    Double clicks on  file QVTRtoXSLT.jar   or input  " java -jar QVTRtoXSLT.jar"   from  the command line,  open the code generator:

    Select the UMLtoDB-transfor.xml for the Input QVT-R model, then push the Load button.  After that, select the transformation umlTordbms to generate, and select the path to output XSLT stylesheet. Press the START button, an XSLT stylesheet umlTOrdbms.xslt will  be generated.  It is a pure XSLT program which can be executed in any XSLT processor.

    Execute the Transformation

    Run the XSLTrunner.jar , open the transformation runner.

    Select the umlTOrdbms.xslt   as the XSLT transformation file,  press the Load button, then select the library.uml as the input source model, select the output target model, finally press the START button to complete the transformation.

    There is another transformation, rdbmsToSQL, in the example transformation model. This transformation produces SQL scripts
    from the RDBMS model. The input of the transformation is the output of the umlTOrdbms.xslt   stylesheet, and it will output a html document.
