Model Transformation with QVTR and XSLT

Multiplicity to OCL transformation

  

The transformation generates OCL class invariants from  the association multiplicities of a class diagram. It uses only one metamodel. Here the rule RootToRoot and rule ClassToClass create the UML Model and Class as context to put the generated OCL constraints. Then rule AssoEndToOCL handles  multiplicity  m..n, where 1<m<n, and m<n<*,  rule  AssoEndToOCL_1 handles  multiplicity  1..1, and rule AssoEndToOCL_No handles  multiplicity  0..*.
