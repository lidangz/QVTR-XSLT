Model Transformation with QVTR and XSLT

UML Activity Diagram to CSP transformation

  

This example is taken from the AGTIVE 2007 Tool Contest.  The transformation definition has 14 rules that can be grouped as follows:

  • CspContainer: the  initial rule that maps for the activity diagram and generates the root node of the CSP model;

  • EdgeToPA: the rule is used to generate a ProcessAssignment for each ActivityEdge node. 

  • ActionToPrefix : the rule generates a Prefix process (starts with an event)  for each Action node;

  • MergeNode: the rule processes the MergeNode node of the activity model; 

  • DecisionNode, DecisionNodeNext, DecisionNodeLast:  three rules are used to process DecisionNodes of the activity model;

  • ForkNode, ForkNodeNext, ForkNodeLast 
    • :  three rules are used to process ForkNodes  of the activity model;

  • JoinNode, JoinNodeGo, JoinNodeSKIP 
    • :  three rules are used to process JoinNodes  of the activity model;

  • FinalToSKIP 
    • :  the rule generates a SKIP for each ActivityEdge that points to a FinalNode .
